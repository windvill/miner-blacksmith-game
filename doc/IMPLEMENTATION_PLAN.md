# 로그인 페이지 및 게임 페이지 분리 구현 계획

기존의 단일 `index.html` 파일로 구성된 **Miner & Blacksmith** 프로젝트를 **로그인 페이지 (`login.html`)**와 **실제 게임 실행 페이지 (`game.html`)**로 분리하고, 정상 실행 여부를 테스트합니다.

---

## 1. 개요 및 배경

현재 `index.html` 상단에 인증 카드(이메일, 비밀번호, 로그인/회원가입 버튼)와 모든 게임 실행 화면(채광, 던전, 대장간, 상점)이 함께 포함되어 있어 화면 구성이 복잡합니다.  
이를 **시작/로그인 랜딩 페이지**와 **전용 게임 플레이 페이지**로 분리하여 사용자 경험(UX)과 소스 구조를 개선합니다.

---

## 2. 주요 변경 사항

### Component 1: 로그인 및 게스트 진입 페이지 (`login.html`) [NEW]
* **디자인 & UI**: 게이밍 감성의 어두운 모던 테마 적용 (Miner & Blacksmith 비주얼 아트 컨셉).
* **기능**:
  * Supabase 이메일/비밀번호 로그인 및 회원가입.
  * 로그인 성공 시 세션을 유지하며 `game.html`로 자동 이동.
  * **"게스트로 시작 (로컬 저장)"** 버튼을 제공하여 로그인 없이 바로 게임 페이지 진입 가능.
  * 이미 로그인된 세션이 있는 경우 즉시 `game.html`로 자동 리다이렉트.

### Component 2: 메인 게임 페이지 (`game.html`) [NEW]
* **디자인 & UI**:
  * 기존 `index.html`의 채광, 던전, 대장간, 인벤토리/상점/기록 UI 그대로 유지.
  * 기존의 커다란 인증 입력 상자 대신, 상단 **플레이어 정보 바(Player Bar)** 배치 (접속 이메일 또는 "게스트 플레이어", 클라우드 저장, 로그아웃 버튼).
* **기능**:
  * 게임의 모든 핵심 기능(채광, 10회 채광, 탐험, 대장간 제련, 상점, Web Audio BGM/SFX) 온전히 작동.
  * 로그인 유저: 진행 상황 자동/수동 Supabase 클라우드 저장.
  * 게스트 유저: LocalStorage 기반 즉시 저장 및 게임 진행.
  * 로그아웃 클릭 시 `login.html`로 이동.

### Component 3: 메인 진입점 (`index.html`) [MODIFY]
* 접속 시 Supabase 세션 유무를 파악하여, 로그인 상태면 `game.html`, 비로그인 상태면 `login.html`로 스마트 리다이렉트하는 Entry Point로 전환.

---

## 3. 검증 및 테스트 계획

### 로컬 웹 서버 실행 및 동작 검증
1. **웹 서버 가동**:
   * 로컬 HTTP 서버(예: `python3 -m http.server 8080`) 실행.
2. **브라우저 테스트 (`browser_subagent`)**:
   * `http://localhost:8080/login.html` 접속 확인.
   * "게스트로 시작" 클릭 후 `game.html`로 정상 이동하는지 확인.
   * `game.html`에서 `⛏ 채광`, `⛏⛏ 10회 채광`, `제작`, `BGM 켜기` 등 핵심 기능이 정상 작동하는지 확인.
   * 결과 화면 스크린샷/비디오 녹화 검증.

---

## 4. 사용자 검토 항목 (User Review Required)

> [!NOTE]
> * 로그인 없이 바로 즐길 수 있도록 **"게스트 모드"**를 기본 탑재합니다.
> * `index.html`에 접속하면 로그인 상태에 따라 `login.html` 또는 `game.html`로 자동 연결되도록 구성됩니다.
